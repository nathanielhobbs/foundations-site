Midterm Practice — Solution Key

1) Multiple Choice

  1. 1.1 # 2. # This is a comment.
    Python single-line comments start with #.
  2. 1.2 # 2.
    elif runs only if all prior if/elif conditions were false and its own condition is true.
  3. 1.3 # 1. 2.5
    5/2 is floating-point division in Python 3 → 2.5.
  4. 1.4 # 1.
    Lists are mutable; tuples are immutable.
  5. 1.5 # 1 and # 3
    The shown output is a single line. (1) a single print with the exact string works; (3) print("This is", name, "and he’s", age, "years old.") matches intent. (2) is wrong: concatenating str with int without str(age) raises a TypeError.

2) True/False

  1. 2.1 False — if doesn’t require an else.
  2. 2.2 False — += is not “by 1” specifically; it adds the RHS.
  3. 2.3 False — Positional parameter order matters (unless keywords are used).
  4. 2.4 True — You can mix required and defaulted parameters.
  5. 2.5 True — *args packs a variable number of args into a tuple.

4) Open-Ended

4.1 Define a function that returns the sum of values at even indices (0,2,4,…).

def sum_even_indices(nums):
    """Assumes nums is a list of exactly 10 integers."""
    return sum(nums[i] for i in range(0, len(nums), 2))
Equivalent answers (e.g., sum(nums[0::2])) earn full credit.

4.2 What does this code do?

def sum(a, b):
    return a + b

print('The sum of 5 and 6 is ' + sum('5', '6'))
print('The sum of 1 and 2 is ' + sum(1, 2))
First line concatenates strings → prints The sum of 5 and 6 is 56. Second line raises TypeError because you’re adding a string to an int. A robust fix: 
print(f"The sum of 5 and 6 is {sum(5, 6)}")
print(f"The sum of 1 and 2 is {sum(1, 2)}")
